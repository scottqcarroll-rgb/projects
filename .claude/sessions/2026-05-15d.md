## Key Technical Detail â€” Why `--dangerously-load-development-channels`

The official `--channels plugin:telegram@claude-plugins-official` requires the plugin installed through Claude's marketplace (`/plugin install`). We installed it manually, so the registry doesn't recognize it. The workaround:

```bash
claude --dangerously-load-development-channels server:telegram
```

This references the MCP server defined in `~/.mcp.json` directly, bypassing the plugin registry.

## Reinstall Instructions (if server is lost)

1. `curl -fsSL https://bun.sh/install | bash && source ~/.bashrc`
2. `sudo apt-get install -y tmux`
3. Download plugin from GitHub: `anthropics/claude-plugins-official` â†’ `external_plugins/telegram/`
   - Copy to `~/.claude/plugins/telegram@claude-plugins-official/`
   - Run `bun install` in that directory
   - Download `.claude-plugin/plugin.json` and `skills/access/SKILL.md` too
4. Create `~/.claude/channels/telegram/.env`:
   ```
   TELEGRAM_BOT_TOKEN=<token from BotFather>
   ```
5. Copy `projects/telegram-channel/mcp.json.example` to `~/.mcp.json` (update path if needed)
6. Copy scripts from `projects/telegram-channel/` to `~/`, chmod +x both
7. Add cron: `(crontab -l; echo "@reboot ~/boot-claude-telegram.sh >> ~/claude-telegram-boot.log 2>&1") | crontab -`
8. Run `~/start-claude-telegram.sh`, DM bot to get pairing code
9. Manually edit `~/.claude/channels/telegram/access.json` to approve pairing and set allowlist
