## Technical Details

### Why Long Polling Works Better
- **Cost:** Completely free (Telegram API, no Anthropic charges)
- **Latency:** Telegram holds connection open 30 seconds waiting for messages, responds instantly when one arrives
- **Old approach:** Short polling every 2-3 seconds would hammer the API
- **This approach:** Single long-poll connection per bot, zero wasted polling

### Bot Credentials Flow
```
~/.mcp.json (TELEGRAM_BOT_TOKEN)
           ↓
telegram-bot.py loads at startup
           ↓
~/projects/email-agent/.env (ANTHROPIC_API_KEY)
           ↓
Creates Anthropic client + long polls Telegram
```

### Agentic Loop (Tool Use)
1. User sends message → bot receives via `getUpdates()`
2. Bot sends to Claude API with `tools=[bash]`
3. If Claude calls bash tool → bot runs command, sends output back to Claude
4. Claude generates final response → bot sends to Telegram
5. Loop continues in tmux (always listening)
